# Documentation index

Keep docs short. Put each fact in one source-of-record file and link to it elsewhere.

| Topic | Doc |
|---|---|
| Bootstrap/install flow | [bootstrap.md](bootstrap.md) |
| Tests, lint, handoff checks | [testing.md](testing.md) |
| Shell load order, shared helpers | [shell-env.md](shell-env.md) |

| Desktop F1 keyboard help | [help.md](help.md) |

## Rules

- `README.md`: human landing page.
- `AGENTS.md`: agent map and invariants.
- Topic docs own details; avoid duplicated tables.
- Link code when behavior matters.
- `docs/help.md` is generated by `scripts/gen-help`.

## Validation

| Change | Check |
|---|---|
| Docs only | Pre-commit, or `make test` when not committing |
| Bootstrap, globals, installers | Above plus `make test-init` |
| Xmonad/window/keybindings | Above plus `make test-ui-snap-window` |
| `docs/help.md` | `DOTFILES="$PWD" scripts/gen-help`, then inspect diff |
| Shell lint/formatting | `make lint` when tools exist |
